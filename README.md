# Plodpai Alert
ระบบแจ้งเตือนภัยพิบัติและนำทางสู่จุดปลอดภัย — โครงงานคอมพิวเตอร์ประเภทซอฟต์แวร์

โรงเรียนหนองหินวิทยาคม อ.หนองหิน จ.เลย

---

## ระบบนี้ทำอะไรได้บ้าง

1. **🔔 แจ้งเตือนแผ่นดินไหวเรียลไทม์** — ดึงข้อมูลจาก USGS Earthquake API ทุก 60 วินาที คำนวณระยะเวลาที่คลื่นสั่นสะเทือนจะมาถึงตำแหน่งผู้ใช้
2. **🧪 โหมดจำลองแผ่นดินไหว (Demo Mode)** — จำลองเหตุการณ์เพื่อสาธิตสด โดยข้อมูลไหลผ่าน pipeline เดียวกับข้อมูลจริง (ต่างกันแค่ label `source: DEMO`)
3. **🌏 โหมดขยายพื้นที่** — ขยายรัศมีค้นหาข้อมูลจริงจาก USGS ให้ครอบคลุมทั่วโลก ใช้พิสูจน์ว่าระบบเชื่อมต่อข้อมูลจริงได้ (ไม่ใช้ตอน demo สดเพราะควบคุมเวลาไม่ได้)
4. **🛡️ จุดปลอดภัยอัจฉริยะ (Smart Routing)** — ดึงจุดปลอดภัย (สนามกีฬา สวนสาธารณะ โรงเรียน โรงพยาบาล สถานีตำรวจ) จาก OpenStreetMap รอบตำแหน่งผู้ใช้ แล้วคำนวณคะแนนแนะนำจาก 4 ปัจจัย (ระยะทาง/ความจุที่เหลือ/ประเภทสถานที่/ความปลอดภัยเส้นทาง) เพื่อกระจายคนไม่ให้ไปกองที่จุดเดียว
5. **📍 เช็คอิน** — เมื่อถึงจุดปลอดภัยแล้ว กดเช็คอินเพื่ออัปเดตความจุแบบเรียลไทม์ ทำให้คนถัดไปได้รับคำแนะนำที่กระจายตัว
6. **🆘 SOS ขอความช่วยเหลือ** — กดค้าง 3 วินาที (กันกดพลาด) เลือกสถานะ (ติดอยู่/บาดเจ็บ/ช่วยผู้อื่น) ส่งพิกัด GPS + ระดับแบตเตอรี่ให้เจ้าหน้าที่ทันที
7. **📊 Dashboard** — หน้าแสดงภาพรวมสำหรับหน่วยกู้ภัย ไม่ต้องล็อกอิน เห็นแผนที่ SOS/จุดปลอดภัย + สถิติเรียลไทม์

## วิธี Deploy ขึ้น GitHub Pages (ฟรี 100%)

1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo `plodpai-alert` ของ GitHub (ผ่านหน้าเว็บ github.com กด "Add file" → "Upload files" ลากไฟล์ทั้งหมดวางแล้วกด Commit — ไม่ต้องใช้ command line ก็ได้)
2. ไปที่ repo → **Settings** → เมนูซ้าย **Pages**
3. หัวข้อ "Build and deployment" → Source เลือก **"Deploy from a branch"**
4. Branch เลือก **`main`** และโฟลเดอร์เลือก **`/ (root)`** → กด **Save**
5. รอ 1-2 นาที จะได้ลิงก์ประมาณ `https://projectcomnwk.github.io/plodpai-alert/`
6. เปิดลิงก์นั้นบนมือถือ (ต้องอนุญาต GPS) ก็ใช้งานได้ทันที

## วิธีทดสอบระบบ (ตามแผนที่ตกลงกันไว้)

**วิธีที่ 1 — Simulate Mode (ใช้ demo สดหน้ากรรมการ):**
กดปุ่ม "🧪 จำลองแผ่นดินไหว" → เลื่อนเลือกขนาด → กด "เริ่มจำลอง" ระบบจะแจ้งเตือนทันทีเหมือนเหตุการณ์จริง ควบคุมเวลาได้ 100%

**วิธีที่ 2 — เก็บเป็นหลักฐานว่าระบบดึงข้อมูลจริงได้:**
กดปุ่ม "🌏 เปิดโหมดขยายพื้นที่" ทิ้งไว้สักครู่ (ทุก 60 วินาทีระบบเช็คซ้ำ) จะจับแผ่นดินไหวขนาดเล็กที่เกิดขึ้นจริงทั่วโลกได้แน่นอน (มีทุกวัน) → อัดวิดีโอ/แคปหน้าจอเก็บไว้เป็นหลักฐานประกอบเอกสารโครงงาน ไม่ใช้วิธีนี้ตอน demo สดเพราะควบคุมเวลาไม่ได้

## ข้อจำกัดที่ควรรู้ (และพูดกับกรรมการได้อย่างมั่นใจ)

- **เสียงแจ้งเตือนแม้ปิดเสียง/ล็อกหน้าจอ** — เป็นสิทธิ์พิเศษที่ Apple/Google ให้เฉพาะระบบแจ้งเตือนภัยพิบัติของรัฐบาล (Cell Broadcast) แอปทั่วไปในโลกเข้าไม่ถึงสิทธิ์นี้ ระบบนี้ใช้ banner + เสียง + สั่นผ่านหน้าเว็บแทน ซึ่งเป็นวิธีที่ดีที่สุดที่ทำได้ในระดับนักพัฒนาทั่วไป
- **ความจุของแต่ละจุดปลอดภัย** เป็นค่าประมาณตามประเภทสถานที่ (สนามกีฬา, สวน ฯลฯ) เนื่องจาก OpenStreetMap ไม่มีข้อมูลนี้ — เป็นจุดที่ต่อยอดได้ด้วยการให้ชุมชนช่วย verify ความจุจริงในอนาคต
- **คะแนน "เส้นทางปลอดภัย"** ตอนนี้เป็นค่าคงที่ (baseline) — ต่อยอดได้ด้วยระบบรายงานถนนพัง/สิ่งกีดขวางจากชุมชน (crowdsource) ตามที่ออกแบบไว้ในแผนงานเดิม
- **Firestore Rules** ปัจจุบันเปิดกว้างพอสมควรเพราะไม่มีระบบล็อกอิน เหมาะกับการสาธิตและใช้งานจริงในวงจำกัด ก่อนขยายใช้งานวงกว้างแนะนำเพิ่ม Firebase App Check

## ต่อยอดได้อย่างไรบ้าง

- เพิ่มภัยพิบัติประเภทอื่น (น้ำท่วม, ไฟป่า) โดยใช้โครงสร้าง Firestore เดิม เปลี่ยนแค่แหล่งข้อมูล (จาก USGS เป็น API พยากรณ์น้ำ/ตรวจจับควันไฟ)
- เพิ่ม Telegram Bot แจ้งเตือนแบบ push (ดูวิธีสร้าง Bot ได้จาก @BotFather ใน Telegram) — ควรทำผ่าน Cloud Function แทนการเรียกจาก client โดยตรง เพื่อไม่ให้ token หลุดในโค้ดสาธารณะ
- ให้ชุมชนช่วย verify/แก้ไขความจุจริงของแต่ละจุดปลอดภัยผ่านฟอร์มเพิ่มเติม
