# Plodpai Alert
ระบบแจ้งเตือนภัยพิบัติและนำทางสู่จุดปลอดภัย — โครงงานคอมพิวเตอร์ประเภทซอฟต์แวร์

โรงเรียนหนองหินวิทยาคม อ.หนองหิน จ.เลย

---

## ระบบนี้ทำอะไรได้บ้าง

1. **🔔 แจ้งเตือนแผ่นดินไหวเรียลไทม์** — ดึงข้อมูลจาก USGS Earthquake API ทุก 60 วินาที พร้อมเสียงไซเรน สั่นเครื่อง และป๊อบอัพเต็มจอ
2. **🧪 โหมดจำลองแผ่นดินไหว (Demo Mode)** — จำลองเหตุการณ์เพื่อสาธิตสด ข้อมูลไหลผ่าน pipeline เดียวกับข้อมูลจริง
3. **🌏 โหมดขยายพื้นที่** — ขยายรัศมีค้นหาข้อมูลจริงจาก USGS ทั่วโลก ใช้พิสูจน์ว่าระบบเชื่อมต่อข้อมูลจริงได้
4. **🛡️ จุดปลอดภัยอัจฉริยะ (Smart Routing)** — ดึงจุดปลอดภัยจาก OpenStreetMap คำนวณ "คะแนนความปลอดภัย" (ตามประเภทสถานที่ + ความจุที่เหลือ) และ "คะแนนแนะนำ" (รวมระยะทางด้วย) เพื่อกระจายคนไม่ให้ไปกองที่จุดเดียว
5. **🧭 นำทางแบบเรียลไทม์** — กด "เดินทางไปจุดนี้" แผนที่จะติดตามตำแหน่งคุณสด พร้อมแถบแสดงระยะทาง/เวลาที่เหลือ
6. **📍 เช็คอินอัตโนมัติ** — เมื่อ GPS ตรวจพบว่าอยู่ใกล้จุดหมายไม่เกิน 40 เมตร จะเด้งป๊อบอัพให้ยืนยันเช็คอินทันที
7. **📍 แจ้งพิกัด/สถานะของตัวเอง** — สำหรับผู้ที่ไม่ได้ไปยังจุดที่ระบบแนะนำ แต่ต้องการรายงานตำแหน่งและสถานะ (ปลอดภัย/ไม่ปลอดภัย/ไม่แน่ใจ)
8. **🆘 SOS ขอความช่วยเหลือ** — กดค้าง 3 วินาที เลือกสถานะ ส่งพิกัด GPS + แบตเตอรี่ให้เจ้าหน้าที่ทันที
9. **🧑‍🤝‍🧑 ระบบข้อมูลผู้ใช้** — กรอกชื่อ-เบอร์โทรครั้งแรกที่ใช้งาน แนบไปกับทุกคำขอ เพื่อให้ทราบว่าใครปลอดภัย/ใครต้องการความช่วยเหลือ
10. **📊 Dashboard สาธารณะ** (`dashboard.html`) — ใครก็เข้าดูได้ ไม่ต้องล็อกอิน แสดง**แค่จำนวนคน**และตำแหน่งแบบไม่ระบุตัวตน **ไม่มีชื่อ/เบอร์โทร**
11. **🔐 Dashboard Admin/หน่วยกู้ภัย** (`admin.html`) — ต้องล็อกอินด้วย Firebase Authentication เห็นข้อมูลครบทุกอย่างรวมชื่อ-เบอร์โทร-พิกัดละเอียด

## โครงสร้างข้อมูล 2 ชั้น (สำคัญมาก — อ่านก่อน deploy)

| คอลเลกชัน | มีชื่อ/เบอร์โทรไหม | ใครอ่านได้ | ใช้ที่ไหน |
|---|---|---|---|
| `sos_alerts` | ✅ มี | เฉพาะ Admin ที่ล็อกอิน | admin.html |
| `safe_requests` | ✅ มี | เฉพาะ Admin ที่ล็อกอิน | admin.html |
| `status_reports` | ✅ มี | เฉพาะ Admin ที่ล็อกอิน | admin.html |
| `public_pins` | ❌ ไม่มี | ใครก็ได้ | dashboard.html (สาธารณะ) |
| `zone_state` | ❌ ไม่มี | ใครก็ได้ | ทั้งสองหน้า |
| `quake_events` | ❌ ไม่มี | ใครก็ได้ | ใช้เก็บประวัติ |

ทุกครั้งที่มีคน SOS / เช็คอิน / แจ้งพิกัดตัวเอง ระบบจะเขียนข้อมูล **2 ชุดพร้อมกัน** โดยใช้ id เดียวกัน:
- ชุดเต็ม (มีชื่อ-เบอร์โทร) → เก็บใน `sos_alerts` / `safe_requests` / `status_reports`
- ชุดสาธารณะ (ไม่มีชื่อ-เบอร์โทร แค่ตำแหน่ง+สถานะ) → เก็บใน `public_pins`

Dashboard สาธารณะอ่านแค่ `public_pins` + `zone_state` เท่านั้น จึงไม่มีทางเห็นชื่อ/เบอร์โทรได้เลยไม่ว่าจะพยายามอย่างไร (ถูกบล็อกด้วย Firestore Rules ไม่ใช่แค่ซ่อนในหน้าเว็บ)

## ⚙️ ขั้นตอนตั้งค่าที่ต้องทำเพิ่ม (สำคัญ — ต้องทำก่อนใช้งาน Admin Dashboard ได้)

### 1. เปิดใช้งาน Firebase Authentication

1. ไปที่ [Firebase Console](https://console.firebase.google.com) → เลือกโปรเจค `plodpai-alert`
2. เมนูซ้าย → **Build** → **Authentication** → กด **Get started**
3. แท็บ **Sign-in method** → เลือก **Email/Password** → เปิด (Enable) → **Save**

### 2. สร้างบัญชี Admin/หน่วยกู้ภัย

1. ในหน้า Authentication → แท็บ **Users** → กด **Add user**
2. กรอกอีเมลและรหัสผ่านของครู/หน่วยกู้ภัยที่จะดูแลระบบ (เช่น `admin@plodpai.local` กับรหัสผ่านที่ตั้งเอง)
3. กด **Add user** — ทำซ้ำได้หลายบัญชีถ้ามีทีมงานหลายคน

**บัญชีนี้ต้องสร้างเองผ่าน Firebase Console เท่านั้น** ไม่มีหน้าสมัครสมาชิกในเว็บ (ตั้งใจออกแบบแบบนี้ เพื่อไม่ให้ใครก็ได้สมัครเป็น Admin เอง)

### 3. อัปเดต Firestore Rules

Firestore Database → แท็บ **Rules** → คัดลอกเนื้อหาจากไฟล์ `firestore.rules` วางแทนของเดิม → กด **Publish**

## คะแนนความปลอดภัยของจุดปลอดภัยคำนวณอย่างไร

ระบบคำนวณ 2 ค่าแยกกันสำหรับแต่ละจุด:

**คะแนนความปลอดภัย (แสดงผลให้ผู้ใช้เห็น)** = 60% ประเภทสถานที่ + 40% ความจุที่ยังว่าง
- ประเภทสถานที่: สนามกีฬา/สวนสาธารณะ = 1.0, สนามโรงเรียน = 0.8, ศูนย์ชุมชน = 0.6, โรงพยาบาล/สถานีตำรวจ = 0.4 (เป็นจุดช่วยเหลือมากกว่าจุดอพยพ)
- ความจุที่ยังว่าง: คนยิ่งน้อยเทียบกับความจุ ยิ่งได้คะแนนสูง

**คะแนนแนะนำ (ใช้จัดอันดับเพื่อเลือกจุดที่ดีที่สุด)** = 40% ระยะทาง + 35% ความจุที่เหลือ + 15% ประเภทสถานที่ + 10% เส้นทางปลอดภัย (baseline คงที่ ต่อยอดได้)

ความจุเป็นค่าประมาณตามประเภทสถานที่ เนื่องจาก OpenStreetMap ไม่มีข้อมูลนี้จริง — เป็นจุดที่ต่อยอดได้ด้วยการให้ชุมชนช่วย verify ความจุจริงในอนาคต

## วิธี Deploy ขึ้น GitHub Pages (ฟรี 100%)

1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo ของ GitHub (ลากไฟล์/โฟลเดอร์ทั้งหมด **ที่อยู่ข้างในโฟลเดอร์นี้** ไม่ใช่ลากทั้งโฟลเดอร์แม่ วางในหน้า "Add file → Upload files")
2. ไปที่ repo → **Settings** → **Pages**
3. Source เลือก **"Deploy from a branch"** → Branch **`main`** → โฟลเดอร์ **`/ (root)`** → **Save**
4. รอ 1-2 นาที จะได้ลิงก์ประมาณ `https://<username>.github.io/<repo>/`

## วิธีทดสอบระบบ

**แจ้งเตือนแผ่นดินไหว:** กด "🧪 จำลองแผ่นดินไหว" สำหรับ demo สด, กด "🌏 เปิดโหมดขยายพื้นที่" เพื่อเก็บหลักฐานว่าระบบดึงข้อมูลจริงจาก USGS ได้ (ไม่ใช้ตอน demo สดเพราะควบคุมเวลาไม่ได้)

**จุดปลอดภัย + นำทาง:** กด "ขอจุดปลอดภัย" → กด "เดินทางไปจุดนี้" → เดินเข้าใกล้จุดหมาย (หรือทดสอบด้วยจุดใกล้ตำแหน่งปัจจุบันมากๆ) → ดูว่าป๊อบอัพเช็คอินเด้งขึ้นเองไหม

**แจ้งพิกัดตัวเอง:** กด "แจ้งพิกัด/สถานะของฉัน" → เลือกสถานะ → เช็คว่าขึ้นบน Dashboard สาธารณะเป็นจุดไม่ระบุชื่อ และขึ้นบน Admin Dashboard พร้อมชื่อ

**Dashboard สาธารณะ vs Admin:** เปิด `dashboard.html` ในเบราว์เซอร์ปกติ (ไม่ล็อกอิน) ควรเห็นแค่ตัวเลข ไม่เห็นชื่อเลย แล้วเปิด `admin.html` ล็อกอินด้วยบัญชีที่สร้างไว้ ควรเห็นชื่อ-เบอร์โทรครบ

## ข้อจำกัดที่ควรรู้ (และพูดกับกรรมการได้อย่างมั่นใจ)

- **เสียงแจ้งเตือนแม้ปิดเสียง/ล็อกหน้าจอ** เป็นสิทธิ์พิเศษเฉพาะ Cell Broadcast ของรัฐบาล แอปทั่วไปเข้าไม่ถึง ระบบนี้ใช้ banner + เสียงสังเคราะห์ + สั่นแทน ซึ่งเป็นวิธีที่ดีที่สุดที่นักพัฒนาทั่วไปทำได้
- **ความจุของแต่ละจุด** เป็นค่าประมาณตามประเภทสถานที่ ต่อยอดได้ด้วยการ verify จากชุมชนจริง
- **บัญชี Admin** ต้องสร้างเองผ่าน Firebase Console ไม่มีหน้าสมัครสมาชิกในเว็บ (ตั้งใจเพื่อความปลอดภัย)
- **Firestore Rules** ปัจจุบันยังไม่มี Firebase App Check ก่อนใช้งานจริงในวงกว้างแนะนำเพิ่มเพื่อกันบอทยิง request ปลอม

## ต่อยอดได้อย่างไรบ้าง

- เพิ่มภัยพิบัติประเภทอื่น (น้ำท่วม, ไฟป่า) โดยใช้โครงสร้าง Firestore เดิม
- เพิ่ม Telegram Bot แจ้งเตือนแบบ push ให้ Admin เมื่อมี SOS ใหม่ (ควรทำผ่าน Cloud Function เพื่อไม่ให้ token หลุด)
- ให้ชุมชนช่วย verify/แก้ไขความจุจริงของแต่ละจุดปลอดภัย
- เพิ่ม Firebase App Check ป้องกันการยิง request ปลอมเข้าคอลเลกชันสาธารณะ
- ผูกคะแนน "เส้นทางปลอดภัย" เข้ากับรายงานถนนพัง/สิ่งกีดขวางจากชุมชนแบบ crowdsource
